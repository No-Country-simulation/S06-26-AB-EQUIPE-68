============================================================
  BiT App — Guia Completo de Execução
============================================================

1. PRÉ-REQUISITOS
------------------------------------------------------------
- Docker Desktop instalado e rodando
- Chave API OpenAI com saldo (billing > $5)

2. CRIAR ARQUIVO .env.production NA RAIZ DO PROJETO
------------------------------------------------------------
Se não existir, crie o arquivo .env.production com:

  DB_URL=jdbc:mysql://bitapp-mysql:3306/db_bitapp?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
  DB_DRIVER=com.mysql.cj.jdbc.Driver
  DB_USERNAME=root
  DB_PASSWORD=123456
  N8N_ASSESSMENT_URL=http://bitapp-n8n:5678/webhook/bit/agent
  N8N_MENTAL_HEALTH_URL=http://bitapp-n8n:5678/webhook/bit/agent
  N8N_TIMEOUT=10000
  N8N_MAX_RETRIES=3
  JWT_SECRET=bitapp-secret-key-change-in-production-512-bits-minimum
  JWT_EXPIRATION=86400000
  JWT_REFRESH_EXPIRATION=604800000
  SPRING_PROFILES_ACTIVE=prod
  OPENAI_API_KEY=<COLE_SUA_CHAVE_AQUI>

3. SUBIR OS CONTAINERS
------------------------------------------------------------
Na raiz do projeto, rode:

  docker-compose up -d --build

Aguarde todos os serviços ficarem healthy:
  - bitapp-mysql   (porta 3307)
  - bitapp-backend (porta 8080)
  - bitapp-n8n     (porta 5679)

4. CONFIGURAR O N8N (primeira vez)
------------------------------------------------------------
Acesse: http://localhost:5679

  a) Setup do owner:
     - Email:    admin@bitapp.com
     - Senha:    BiT@2026!
     - Nome:     Admin BiT

  b) Criar credencial OpenAI:
     - Menu Settings > Credentials > Add Credential
     - Busque "OpenAI API"
     - Cole sua API Key
     - Salve como "OpenAI API"

  c) Importar workflow:
     - Menu Workflows > Import from File
     - Selecione: n8n/bitapp-agent-workflow.json
     - Após importar, abra o workflow
     - Nos 3 nodes OpenAI (Classificar Intent,
       Assessment Agent, Mental Health Agent):
       - Clique no node > Credentials > selecione "OpenAI API"
     - Clique em "Save"
     - Ative o workflow (toggle Active = ON)

5. VERIFICAR SE TUDO ESTÁ RODANDO
------------------------------------------------------------
  docker-compose ps

Todos devem mostrar "Up" e "healthy".

6. TESTAR O SISTEMA
------------------------------------------------------------
  a) Backend (API):
     Abrir no navegador ou Postman:
     http://localhost:8080/actuator/health

  b) Frontend:
     Abrir o index.html na pasta frontend/ no navegador
     ou subir com Live Server (porta 3000/5173)

  c) n8n Editor:
     http://localhost:5679
     Login: admin@bitapp.com / BiT@2026!

  d) Testar webhook n8n (via terminal):
     curl -X POST http://localhost:5679/webhook/bit/agent ^
       -H "Content-Type: application/json" ^
       -d "{\"nome\":\"Teste\",\"idade\":25,\"hardSkills\":[\"Java\"],\"softSkills\":[\"Comunicacao\"],\"tecnologias\":[\"Spring Boot\"],\"tipo\":\"assessment\"}"

8. COMANDOS UTEIS
------------------------------------------------------------
  Reiniciar tudo:          docker-compose restart
  Ver logs n8n:            docker-compose logs -f n8n
  Ver logs backend:        docker-compose logs -f backend
  Parar tudo:              docker-compose down
  Rebuild completo:        docker-compose up -d --build --force-recreate
  Limpar dados n8n:        docker-compose down -v (apaga volume n8n)

9. TROUBLESHOOTING
------------------------------------------------------------
  PROBLEMA: n8n mostra "Wrong username or password"
  SOLUÇÃO:  O N8N_BASIC_AUTH foi removido. Use:
            admin@bitapp.com / BiT@2026!

  PROBLEMA: Erro 429 da OpenAI
  SOLUÇÃO:  Sua API key sem saldo. Adicione créditos em
            https://platform.openai.com > Billing

  PROBLEMA: Webhook retorna 404
  SOLUÇÃO:  Verifique se o workflow está ativo no n8n.
            O path deve ser: /webhook/bit/agent

  PROBLEMA: Backend não conecta no MySQL
  SOLUÇÃO:  Aguarde o MySQL ficar healthy antes de subir:
            docker-compose up -d mysql
            aguarde healthy...
            docker-compose up -d

  PROBLEMA: Frontend não carrega dados
  SOLUÇÃO:  Verifique se o backend está rodando na porta 8080
            e se o CORS permite sua origem.

10. FLUXO DA APLICAÇÃO
------------------------------------------------------------
  Frontend (HTML/JS)
       |
       | POST /api/assessment ou /api/mental-health
       v
  Backend (Spring Boot - porta 8080)
       |
       | POST http://bitapp-n8n:5678/webhook/bit/agent
       v
  n8n Workflow (GPT-4o-mini)
       |
       | Classifica intent -> Assessment Agent ou Mental Health Agent
       | Retorna JSON no formato do DTO
       v
  Backend persiste no MySQL e retorna ao Frontend
